# Task

Для клиента:

- Приложение должно иметь форму регистрации с полями "Имя", "Фамилия", "Номер телефона", "Почта".

- Приложение должно иметь форму аутентификация и авторизации (права доступа владелец/арендатор/субарендатор). При успешной аутентификации пользователь переходит в личный профиль, в котором отображена вся его история посещений салона (мастера). Пользователь должен иметь возможность зарегистрироваться из любого приложения которым он пользуется (Instagram, Facebook...). Там же должна быть продублирована кнопка "Записаться".

- В приложении должен быть перечень услуг с актуальной стоимостью и продолжительностью по времени. Должно быть уточнение, что стоимость может варьироваться в зависимости от расхода краски и других расходных материалов.

- Приложение должно быть портфолио работ мастера.

- На главной странице должна иметься кнопка "Записаться". При нажатии на которую отображается календарь (на месяц или более) с отображением свободного времени для записи. Так же должна быть возможность отменить запись.

- В приложении должна быть страница с контактами данного мастера/салона.

Для мастера:

- Список клиентов. При нажатии на имя клиента, будет совершён переход на его историю посещений.

- Расписание в виде календаря с возможностью записать клиента. Так же должна быть возможность отменить запись.

 - Оплаты. Оплата может осуществляться несколькими путями:

 а) оплата наличными;
 б) оплата картой (следует реализовать логику для возможности оплаты через терминал);
 в) чаевые.

 - Расходные материалы. Должны иметься поле, в котором записывается купленные расходные материалы. Поле, в котором записываются проданные косметические средства.
